  • Data Types

    1. primitive data types
      • number string boolean
      • undefined null:undefined and null are equal in value but different in type:
      • symbol bigint
    2. Object
      • Objects are mutable: They are addressed by reference, not by value.
      • Create new objects:
        • Object Literal: const person = {firstName:"John", lastName:"Doe", age:50, eyeColor:"blue"};
        • JavaScript Keyword new: const person = new Object();
        • Create an object using Object.create().
      • In JavaScript, almost "everything" is an object. Booleans can be objects (if defined with the new keyword)
      • Properties: Properties are the values associated with a JavaScript object. Accessing JavaScript Properties:
        objectName.property // person.age
        objectName["property"] // person["age"]
        objectName[expression] // x = "age"; person[x]
      • Methods: actions that can be performed on objects.
        Built-In Methods:
      • Display Objects:
        1. Using a Loop: for (let x in person) { txt += person[x] + " "; };
        2. Object.values(): all values converted to an array
        3. JSON.stringify(): converts dates into strings; but will not stringify functions. It is also possible to stringify JavaScript arrays.
      • Object Accessors
        Getters
        1. get fullName() { return this.firstName + " " + this.lastName; }
        2. access as a property: person.fullName.
        Setters
        1. set lang(lang) { this.language = lang.toUpperCase(); }
        2. Set an object property: person.lang = "en";
    3. The typeof Operator
      • typeof operator always returns a string
      • return value: string number boolean undefined function object
    4. The constructor Property:The constructor property returns the constructor function for all JavaScript variables.
    5. Check the constructor property to find out the type of an object:
      for example: myArray.constructor.toString().indexOf("Array") > -1;
    typeof "John" // Returns "string"
    typeof 3.14 // Returns "number"
    typeof NaN // Returns "number"
    typeof myCar // Returns "undefined"
    typeof false // Returns "boolean"
    typeof [1,2,3,4] // Returns "object"
    typeof {name:'John', age:34} // Returns "object"
    typeof new Date() // Returns "object"
    typeof function () {} // Returns "function"
    typeof null // Returns "object"
    "John".constructor // Returns function String() {[native code]}
    (3.14).constructor // Returns function Number() {[native code]}
    false.constructor // Returns function Boolean() {[native code]}
    [1,2,3,4].constructor // Returns function Array() {[native code]}
    {name:'John',age:34}.constructor // Returns function Object() {[native code]}
    new Date().constructor // Returns function Date() {[native code]}
    function () {}.constructor // Returns function Function(){[native code]}

  • Literal & new

    1. Use string literals "" instead of new String().
    2. Use number literals 50 instead of new Number().
    3. Use boolean literals true / false instead of new Boolean().
    4. Use object literals {} instead of new Object().
    5. Use array literals [] instead of new Array().
    6. Use pattern literals /()/ instead of new RegExp().
    7. Use function expressions () {} instead of new Function().
