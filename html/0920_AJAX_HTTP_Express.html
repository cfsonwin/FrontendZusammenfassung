  • HTTP

    1. consist of Request and Respond
    2. Request
      • A start-line describing the requests to be implemented, or its status of whether successful or a failure. This start-line is always a single line. 包含HTTP method:get还是post等等->request target:url路径->HTTP version
      • An optional set of HTTP headers specifying the request, or describing the body included in the message. in form of [key: value]
      • A blank line indicating all meta-information for the request has been sent.
      • An optional body containing data associated with the request (like content of an HTML form), or the document associated with a response. The presence of the body and its size is specified by the start-line and HTTP headers.
    3. Response
      • Status line: protocol version->status code[202/404/403]->status text["OK"]
      • Headers
      • blank line
      • Body

  • Express

    1. npm init --yes
    2. install with nodeJs: npm i express
    3. simple express.js:
      // invote Express frame
      const { request, response } = require("express");
      const express = require("express");

      // create express app-obj
      const app = express();

      // create router rules
      app.get("/", (request, response)=>{
      response.send("hallo express");
      });

      // start service
      app.listen(8000, ()=>{
      console.log("Service Started!");
      });
    4. start service: node ./express.js
    5. allow self-define headers: response.setHeader("Access-Control-Allow-Headers", "*");
