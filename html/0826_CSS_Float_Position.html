  • Float

    1. 使元素向左或向右移动
    2. value=none / left / right
    3. 元素使用float后便不需要满足水平布局的等式
    4. 使用float后会从文档流中脱离，不占位置
    5. 浮动元素不会覆盖其他元素，形成水平布局，一行容纳不下就会自动换行
    6. 浮动元素不会覆盖文字，文字自动环绕
    7. 从文档流中脱离后，块儿元素不独占一行高度宽度默认被内容撑开，inline可以设置宽高
    3/4水平并列，2的位置不会因为1移动，div和span不设置宽高时都会包裹元素

  • Float问题：高度塌陷

    如果父元素没有设置高度，且子元素设置float，因为其不在文档流之中，父元素高度不能被撑开

      解决办法：
    1. 开启BFC（block formatting context）开启方法：
      I.设置父元素为float
      II.设置为inline-block
      III.设置overflow为非visible
    2. 设置css样式，::after{content:"";clear:both;display:block;}
    3. 同理对于外边距重合问题，可以添加
      ::before, ::after{content:""; display:table; clear:both}

  • Position [top left bottom right]

    1. relativ: 原点相对于其在文档流中的位置，不会脱离文档流，不改变inline/block属性
    2. absolute：相对于containing box（最近的开启定位的祖先元素）定位
      一般会给绝对定位的父元素开启相对定位
      元素位置不变
      脱离文档流：宽度被内容撑开，不占位置
    3. fixed：类似于absolute，位置相对于viewpoint
    4. sticky：兼容性不好，类似于相对定位，在页面滚动时当元素到达一个位置后就会停止移动
    开启绝对定位后，水平布局等式加上left和right
    width auto
    margin:auto
    margin:auto left/right:0;
    margin:auto left/right/bottom/top:0;

  • 元素层级

    1. 绝对定位会提升层级
    2. 对开启了定位的元素，可以通过设置z-index一个整数参数，越大越在上面
    3. 祖先元素层级再高也不会盖住后代元素
